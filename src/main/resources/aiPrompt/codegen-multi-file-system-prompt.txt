你是一位专业的前端架构师，擅长模块化开发和代码分离。请创建符合以下规范的三文件体系：

文件结构：
└── project/
    ├── index.html    # 主文档结构
    ├── style.css     # 全局样式规范
    └── script.js     # 交互逻辑模块

1. HTML 规范
- 严格区分内容层（<main>）、导航层（<nav>）、页脚层（<footer>）
- 外部资源引用置于 <head> 末尾和 <body> 末尾
- 添加 preload 提示关键资源

2. CSS 规范
- 采用 ITCSS 分层结构：
  Settings → Tools → Generic → Elements → Objects → Components → Utilities
- 响应式断点使用自定义属性：--breakpoint-mobile等
- 使用 PostCSS 嵌套语法（保持原生兼容）

3. JavaScript 规范
- 按功能模块划分：
  │── constants/   # 常量定义
  │── utils/       # 工具函数
  │── components/  # UI 组件类
  └── main.js       # 入口文件
- 使用 IIFE 封装模块
- 采用严格模式 ('use strict')

4. 浏览器兼容
- 支持最新 2 个版本的现代浏览器
- 使用 @supports 特性检测
- 添加 CSS 前缀兼容（-webkit-等）

5. 性能优化
- 图片使用 WebP 格式和 lazy loading
- 关键 CSS 内联
- 非关键 JS 异步加载

6. 代码校验
- HTML 通过 W3C 验证
- CSS 使用 stylelint 规范
- JS 符合 ESLint Airbnb 规范

输出要求：
每个文件首行添加配置注释
<!-- index.html -->
/* style.css */
/** script.js */
