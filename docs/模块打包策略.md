# 模块打包策略

### 只需要上传有启动类的应用模块的可执行 jar 包。 

✅ 上传：rich-code-weaver-generator-xxx.jar（主应用）

✅ 上传：rich-code-weaver-user-xxx.jar

✅ 上传：rich-code-weaver-file-xxx-exec.jar（注意是 exec 后缀的）

✅ 上传：rich-code-weaver-prompt-xxx.jar

✅ 上传：rich-code-weaver-social-xxx.jar

✅ 上传：rich-code-weaver-server-admin-xxx.jar

而库模块的 jar 已经被打包到应用模块的可执行 jar 中了（通过 Maven 依赖传递）。

❌ 不需要上传：rich-code-weaver-ai-xxx.jar

❌ 不需要上传：rich-code-weaver-client-xxx.jar

❌ 不需要上传：rich-code-weaver-common-xxx.jar

❌ 不需要上传：rich-code-weaver-model-xxx.jar

### 注意事项：

因为 rich-code-weaver-user 模块依赖了 rich-code-weaver-file 模块，但是在 Maven 构建时，file 模块需要先被编译和安装。   

所以我添加了 <classifier>exec</classifier>，它会生成两个 jar：   
rich-code-weaver-file-1.0-SNAPSHOT.jar（普通 jar，供其他模块依赖）   
rich-code-weaver-file-1.0-SNAPSHOT-exec.jar（可执行 jar，用于部署）   

**上传时选择 -exec.jar 后缀的。**
