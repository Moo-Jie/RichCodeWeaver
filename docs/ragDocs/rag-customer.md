# RichCodeWeaver 平台能力总览

## 平台定位
RichCodeWeaver 是一个围绕数字产物生成、管理、部署、协作与社区互动的 AI 平台。

它的核心目标是：让普通用户通过自然语言描述需求，快速创建数字产物，并在后续通过对话、预览和编辑能力持续完善成果。

## 核心能力
1. 用户可以通过主页输入需求来创建数字产物。
2. 平台支持围绕已有数字产物进行多轮对话式修改。
3. 支持在应用模式中预览成果，并进行可视化编辑。
4. 支持查看热门产物、我的产物、我的收藏、我的素材。
5. 支持部署数字产物、访问部署结果、下载代码压缩包。
6. 支持点赞、收藏、转发、评论等社交互动。
7. 支持用户编辑个人资料，包括昵称、头像、身份、行业、手机号、邮箱等信息。
8. 支持用户作为主创或协作者参与数字产物协作。

## 功能理解主线
- 平台围绕“创建数字产物 -> 继续对话修改 -> 预览成果 -> 可视化编辑 -> 部署 / 下载 -> 分享互动 / 协作”展开。
- AI 客服回答时，应优先把用户的问题归类到这个主线中的某一环节，再给出入口和操作步骤。
- 并不是所有功能都在主页完成。主页更偏向“开始创建”，已有数字产物的继续修改、部署、下载、预览，通常发生在数字产物对话页或对应页面中。

## 普通用户可见页面与入口

### 1. 主页
- 主页是普通用户创建数字产物的主要入口。
- 用户可在主页看到品牌欢迎区、热门数字产物、推荐模板、底部创建输入框，以及右侧 AI 客服。
- 在主页输入需求后，可以创建新的数字产物。

### 2. 我的产物
- 用户可以查看自己创建的数字产物。
- 可以搜索、排序、进入对话、查看部署状态。
- 对于自己拥有的数字产物，可以继续编辑或删除。
- 如果是协作参与的数字产物，通常只能以协作身份进入，不一定拥有全部编辑权限。

### 3. 热门产物
- 用户可以浏览社区热门内容与精选推荐。
- 可以查看产物名称、创建者、互动数据和部署状态。
- 点击卡片后可以进入对应数字产物对话页或详情体验。

### 4. 我的收藏
- 用户可以查看自己收藏过的数字产物。
- 可以取消收藏，并再次进入对应数字产物查看。

### 5. 我的素材
- 用户可以上传、创建、编辑和删除自己的素材。
- 素材类型包括图片、文本、视频、音频、链接等。
- 创建数字产物前，用户可以先选择素材，再把素材带入创建流程。

### 6. 个人中心
- 用户可以查看并编辑昵称、简介、头像、身份、行业。
- 用户可以修改密码、绑定或更换手机号、绑定或更换邮箱。

### 7. 数字产物对话页
- 用户可以在对话模式下继续描述修改需求。
- 用户可以查看历史消息、AI 生成结果和任务进展。
- 用户可以切换到应用模式查看预览。
- 若有权限，用户可进行可视化编辑、下载代码、部署网站、访问部署站点。

## 常见目标与对应入口
- 如果用户想“新建一个数字产物”，应优先引导去主页输入需求。
- 如果用户想“继续修改已经创建的数字产物”，应优先引导去“我的产物”或相关数字产物对话页。
- 如果用户想“查看别人做的热门内容”，应优先引导去“热门产物”。
- 如果用户想“再次进入收藏过的数字产物”，应优先引导去“我的收藏”。
- 如果用户想“先准备图片、文本、链接等素材再创建”，应优先引导去“我的素材”。
- 如果用户想“补充身份、行业、头像、手机号、邮箱等信息”，应优先引导去“个人中心”。
- 如果用户想“查看生成结果、继续提修改需求、部署或下载”，应优先引导去具体数字产物的对话页。

## 主页创建流程
普通用户在主页通常按以下顺序使用：

1. 在输入框中描述想要创建的数字产物。
2. 按需选择创建模式。
3. 如有需要，可先选择素材，或使用推荐模板辅助补全需求。
4. 提交后系统会创建一个新的数字产物，并进入该数字产物的对话页。
5. 系统会基于初始提示词开始生成，用户可在对话页继续追加修改需求。

## 创建模式说明

### 1. 系统分步执行
- 更适合希望结果更完整、更稳妥的场景。
- 系统会按既定流程逐步分析、生成、检查和输出结果。
- 一般更偏重完整性与稳定性。

### 2. Agent 智能生成
- 更适合希望生成过程更灵活、更直接的场景。
- 系统会以更自主的方式完成分析与生成。
- 一般更偏重灵活性与响应速度。

### 3. 用户如何选择
- 如果你希望流程更稳、更适合复杂需求，可优先选择“系统分步执行”。
- 如果你希望先快速得到一版结果，再继续迭代，可优先选择“Agent 智能生成”。

## 用户操作前置条件
- 大多数创建、收藏、素材管理、个人资料修改、继续对话等操作，都以用户已登录为前提。
- 推荐模板的精准度会受到用户身份和行业信息影响，但不代表未完善资料就完全无法创建数字产物。
- 想继续编辑某个数字产物，前提通常是该数字产物属于自己，或者自己具备协作编辑权限。
- 想进行部署，前提通常是数字产物已经成功生成到可部署状态。
- 想使用可视化编辑，前提通常是当前数字产物具备编辑权限，且页面已有可操作成果。

## 数字产物生成与修改逻辑

### 1. 创建后不是一次性结束
- 数字产物不是“提交一次就永远固定”。
- 用户可以在数字产物对话页继续追加需求，逐步修改已有成果。

### 2. 对话模式与应用模式
- 对话模式：更适合持续提需求、查看 AI 回复、追加修改。
- 应用模式：更适合直接查看当前成果预览。

### 3. 可视化编辑
- 在有编辑权限的前提下，用户可以进入可视化编辑。
- 用户可先选中页面元素，再输入修改需求，系统会根据选中的元素上下文完成修改。

### 4. 生成中断与恢复
- 平台支持流式生成。
- 如果网络暂时中断，系统会尝试恢复流式内容。
- 用户刷新页面后，如果上一次任务未完成，系统可能继续恢复未完成的生成状态。

## 推荐模板与身份信息
- 平台会根据用户身份与行业，为用户推荐更匹配的模板。
- 用户未完善身份和行业信息时，推荐模板可能不够精准。
- 用户可以在个人中心补充身份和行业信息，以获取更合适的推荐模板。

## 部署与下载

### 1. 部署能力
- 平台支持将数字产物部署为可访问的网站。
- 部署成功后，用户可以获得站点访问地址，并可直接打开访问。

### 2. 下载代码
- 用户可以下载当前数字产物的代码压缩包。
- 下载适合做本地备份、继续开发或二次使用。

### 3. 重新部署
- 已部署的数字产物可以再次部署，以更新线上内容。

## 社交互动能力
- 用户可以对数字产物点赞、收藏、转发、评论。
- 热门产物页会展示社区热度相关信息。
- 收藏后的产物可在“我的收藏”中再次查看。

## 协作能力
- 平台支持多人围绕同一个数字产物协作。
- 主创可以邀请协作者参与。
- 协作者是否能编辑，取决于当前协作关系与权限状态。
- 如果用户没有编辑权限，就不能直接修改对应数字产物内容。

## 普通用户可直接做的事
1. 在主页输入需求，创建新的数字产物。
2. 进入已有数字产物对话页，继续追加修改需求。
3. 查看预览、切换对话模式与应用模式。
4. 在有权限时进行可视化编辑、部署、重新部署、下载代码。
5. 浏览热门产物、收藏产物、管理个人素材、完善个人资料。
6. 对数字产物进行点赞、收藏、转发、评论等互动。

## 普通用户不能直接做的事
1. 不能查看或操作管理员后台。
2. 不能维护系统提示词、RAG 知识库、模型配置、内部服务配置。
3. 不能跳过权限限制，直接编辑无权编辑的数字产物。
4. 不能在 AI 客服对话中要求系统替自己完成后台配置、内部修复或管理员操作。

## 需要平台维护方处理的事项
- 后台配置错误、系统内部服务异常、模型服务异常、知识库维护、提示词维护，属于平台内部处理范围。
- 普通用户遇到这类问题时，AI 客服可以建议先完成用户侧自查，再提示联系平台维护方，但不能展开内部处理细节。

## AI 客服回答边界

### 1. 必须从普通用户视角回答
- 只回答普通用户可见、可操作、可理解的内容。
- 重点描述页面入口、按钮位置、操作顺序、结果反馈。

### 2. 不应透露后台管理信息
- 不介绍管理员菜单、后台入口、系统维护路径、系统配置、数据库、内部服务。
- 若用户追问后台内容，只说明这是平台内部维护范围，普通用户无需处理。

### 3. 不能虚构平台没有的能力
- 不能编造不存在的页面、按钮、入口或自动处理结果。
- 不能承诺已经替用户完成部署、修改、恢复、审核或配置。

### 4. 超出范围时的处理方式
- 如果问题属于内部维护范围，应明确说明这不是普通用户侧需要处理的内容。
- 如果问题缺少关键信息，应先追问当前页面、当前操作、是否已登录、是否有权限、报错现象等，再回答。

## AI 客服回答优先级
1. 先判断用户问题属于“创建、修改、预览、编辑、部署、下载、互动、协作、个人资料、素材、异常排查”中的哪一类。
2. 如果用户问“在哪”，优先回答页面入口和路径。
3. 如果用户问“怎么做”，优先回答操作步骤，不要只讲概念。
4. 如果用户问“为什么失败”，优先给普通用户能自行检查的事项，再说明下一步建议。
5. 如果用户问“能不能做”，应明确区分：平台支持、当前账号是否有权限、当前页面是否能直接完成、是否需要前置条件。
6. 如果问题超出知识范围，不要猜测，应直接说明暂时无法确认。

## 平台能力边界

### 平台能做的事
1. 根据用户描述创建数字产物。
2. 继续围绕已有数字产物进行多轮修改。
3. 提供预览、编辑、下载、部署、互动与协作等能力。
4. 为普通用户提供模板、素材、收藏、热门内容和个人资料管理。

### 平台不能直接替用户完成的事
1. 不能替用户跳过权限校验。
2. 不能在用户无权限时直接修改别人的数字产物。
3. 不能在客服对话中代替用户完成后台管理或内部配置。
4. 不能保证所有异常都由用户侧立即解决，有些问题需要平台维护方处理。

## 高频混淆场景

### 1. 能看到数字产物，不等于一定能编辑
- 用户可能能浏览、收藏、访问某个数字产物，但不代表一定拥有编辑权限。
- 是否能继续修改，取决于归属关系或协作权限。

### 2. 收藏过数字产物，不等于拥有它
- “我的收藏”只表示用户收藏了该数字产物。
- 收藏不会自动带来编辑、部署或删除权限。

### 3. 推荐模板不够精准，不等于系统坏了
- 推荐模板会受到用户身份和行业信息影响。
- 如果资料不完整，推荐可能不够贴合，但通常不影响基础创建能力。

### 4. 能访问部署站点，不等于一定能修改线上内容
- 访问部署结果只表示可以打开站点。
- 想更新线上内容，通常仍需要先修改数字产物并重新部署。

### 5. AI 客服提供的是指引，不是代执行
- AI 客服可以告诉用户去哪里、怎么做、先检查什么。
- AI 客服不能替用户点击页面、修改后台、修复内部系统或直接完成部署。

## 不应回答或不应展开的内容
- 不应介绍管理员菜单、后台管理入口、内部系统配置页面。
- 不应展开数据库、网关、Nacos、模型参数、服务编排等内部实现。
- 不应给出提示词维护、RAG 知识库维护、后台审核或内部数据处理的操作方法。
- 不应编造“系统已经帮你修复”“已经替你部署成功”“已经替你恢复数据”等结果。

## 常见问题排查思路

### 1. 无法创建数字产物
- 先检查是否已经登录。
- 再检查输入内容是否为空或过于简短。
- 如仍失败，再检查网络连接并稍后重试。

### 2. 没看到推荐模板
- 先检查是否已登录。
- 再检查个人中心中的身份和行业是否已完善。

### 3. 无法继续编辑数字产物
- 先确认当前数字产物是否属于自己，或你是否已成为可编辑协作者。
- 若没有编辑权限，只能先获得对应权限后再操作。

### 4. 生成中途中断
- 先等待系统是否自动恢复。
- 若仍未恢复，可刷新页面后重新查看该数字产物对话页。
- 若多次失败，再联系平台维护方处理。

### 5. 部署失败或无法访问部署结果
- 先确认数字产物是否已经成功生成完成。
- 再尝试重新部署。
- 若仍不可用，再联系平台维护方排查。
