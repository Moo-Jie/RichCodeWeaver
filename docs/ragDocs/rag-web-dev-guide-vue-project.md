# 网页开发规范与建议-Vue项目版本

本文档是 Vue 项目工程模式下的网页开发知识库，供 AI 代码生成时参考。本模式使用 Vue 3 + Vite 构建完整的前端工程项目，AI
具备文件操作工具和素材搜集工具。

## 一、运行环境与部署限制

本系统的最终产物是纯静态文件（构建后的 dist 目录），部署在静态文件服务器上，没有任何后端服务支撑。这意味着：

- 没有 Node.js 服务端运行环境（构建阶段有，运行阶段没有）
- 没有数据库（MySQL、MongoDB、Redis 等均不可用）
- 没有服务端 API 可供调用
- 没有服务端会话管理（Session）
- 没有服务端文件读写能力

所有功能必须在浏览器端独立完成。生成的代码必须通过 `npm run build` 成功构建，构建产物必须能在任意子路径下正常部署。

## 二、数据存储方案

由于没有后端和数据库，所有需要持久化的数据必须使用浏览器本地存储。

### 推荐使用 localStorage

适用于用户偏好设置、表单草稿、购物车数据、待办事项列表、主题切换状态、阅读进度等场景。

使用要点：

- 存储前用 `JSON.stringify()` 序列化，读取后用 `JSON.parse()` 反序列化
- 页面加载时主动从 localStorage 读取数据并还原状态
- 数据变更时及时同步写入 localStorage
- 对 `JSON.parse()` 进行 try-catch 保护，防止数据损坏导致页面崩溃
- 存储键名加上项目前缀，避免与其他网站冲突

### sessionStorage 的适用场景

适用于仅在当前浏览器标签页生命周期内有效的临时数据，如多步骤表单的中间状态、页面间临时传参。

### 不可使用的存储方式

- 禁止调用后端接口保存数据
- 禁止使用 IndexedDB 存储大量结构化数据（复杂度过高，容易出错）
- 禁止使用 Cookie 做数据持久化（容量限制 4KB，且用途不符）

## 三、功能边界——做得到与做不到

### 可以实现的功能

- 页面路由与导航（Vue Router，hash 模式）
- 响应式页面布局（桌面、平板、手机自适应）
- 表单收集与本地展示（不含提交到服务器）
- 前端数据筛选、排序、分页（基于本地数据）
- 动画和过渡效果（CSS 动画、Vue Transition）
- 图表展示（如使用 CDN 引入 Chart.js 等无需 npm 的库）
- 倒计时、计时器、时钟等时间类功能
- 模拟数据的增删改查（数据存在 localStorage 中）
- 暗色模式/主题切换
- 多语言切换（前端硬编码翻译文本）
- 图片轮播、画廊、灯箱效果
- Markdown 渲染（可用 CDN 引入 marked.js）
- 拖拽排序（原生 Drag API 或轻量库）
- 导出功能（导出为 CSV、TXT 等纯文本格式）

### 不可实现的功能（禁止尝试）

- 用户注册/登录的数据持久化（无法将账号密码存在服务端）
- 发送邮件或短信通知
- 文件上传到服务器
- 调用需要 API Key 的第三方服务（如 Google Maps API、OpenAI API、支付接口），除非用户明确提供了 Key
- 服务端渲染（SSR）
- WebSocket 实时通信（无服务端）
- 多用户数据共享或同步
- 后端鉴权与权限管理
- 数据库查询

### 处理原则

当用户需求中包含无法实现的功能时，**不要留下空的函数体、TODO 注释或假接口调用**。正确做法：

- 用 localStorage 模拟数据持久化，提供完整的本地增删改查体验
- 登录功能可以做成前端模拟版本：将用户信息存入 localStorage，页面刷新后自动读取登录状态
- 表单提交改为本地保存并弹出成功提示，而非发送 HTTP 请求
- 需要后端的搜索功能改为前端模糊匹配过滤

## 四、Vue 项目模式的工具能力

本模式下 AI 具备以下工具，可以创建完整的多文件项目结构：

### 文件操作工具

- creatAndWrite：创建新文件并写入内容
- modifyFile：局部修改现有文件
- deleteFile：删除文件
- readDir：读取目录结构
- readFile：读取文件内容

### 素材搜集工具

- searchImages：搜索网络图片
- aiGeneratorImage：AI 生成图片
- searchWeb：联网搜索信息
- scrapeWebPage：抓取网页内容

当系统已提供素材资源时，禁止再调用素材搜集工具。仅在系统未提供任何素材时使用这些工具获取素材。

## 五、第三方依赖管理

### 推荐方式

Vue 项目通过 package.json 管理依赖。可以使用的 npm 包需满足：

- 是纯前端库，不依赖 Node.js 运行时
- 不需要 API Key 或后端服务配合
- 包体积合理，不会导致构建产物过大

### 可安全使用的依赖示例

- vue（核心框架，必装）
- vue-router（路由，必装）
- dayjs（日期处理）
- animate.css（动画库）
- swiper（轮播组件）

### 禁止使用的依赖

- 状态管理库（Pinia、Vuex）——增加复杂度，本项目规模不需要
- TypeScript——增加配置复杂度，容易引发类型错误导致构建失败
- 代码格式化工具（ESLint、Prettier）——与代码生成无关
- 需要后端配合的库（axios 用于调用不存在的后端、socket.io 等）

## 六、图片与静态资源

### 禁止引用未创建的本地资源

这是 Vue 项目构建失败的最常见原因。禁止在代码中 import 或引用 `src/assets/` 下未实际创建的文件。

错误写法：

- `import logo from '@/assets/logo.png'`（文件不存在，构建报错）
- `<img src="/src/assets/banner.jpg">`（文件不存在，构建报错）

正确做法：

- 使用系统提供的素材图片 URL
- 使用在线占位图服务 picsum.photos
- 使用工具搜索或生成图片后获取 URL
- 图标使用内联 SVG 或 Font Awesome CDN

### 图片使用优先级

1. 优先使用系统提供的素材图片 URL（必须全部使用，不可遗漏）
2. 使用工具搜索或生成的图片 URL
3. 使用 picsum.photos 在线占位图补充

## 七、路由配置

必须使用 hash 模式路由，确保构建后的项目在任意子路径下正常工作。

原因：静态文件服务器无法处理 HTML5 History 模式的路由回退。使用 hash 模式（URL 中带 `#`）可以避免页面刷新 404 问题。

## 八、样式与布局

### 响应式设计

所有页面必须适配三种屏幕尺寸：

- 手机：宽度 < 768px
- 平板：768px - 1024px
- 桌面：> 1024px

使用原生 CSS 实现响应式，不引入 UI 框架（减少构建体积和依赖冲突风险）。

### 样式编写原则

- 使用 CSS 变量定义主题色和关键尺寸，便于全局调整
- 优先使用 Flexbox 和 Grid 布局
- 避免使用 `!important` 声明
- `<style scoped>` 确保组件样式隔离

## 九、代码质量红线

### 禁止留下半成品

- 禁止出现 `// TODO` 注释
- 禁止出现空的函数体或方法
- 禁止出现 `console.log` 调试代码
- 禁止出现注释掉的代码块
- 禁止出现 `placeholder` 样式的临时内容

### 确保可运行

- 所有组件的 import 路径必须正确
- 路由配置中引用的页面组件必须已创建
- 事件处理函数必须有实际实现
- 表单交互必须有反馈（成功提示、数据变更等）

### 内容真实性

- 所有文本使用真实、有意义的中文内容
- 产品数据使用接近真实的模拟数据（名称、价格、描述等）
- 禁止使用 "示例文字"、"这里是描述"、"项目1" 等占位内容

## 十、构建与部署验证

Vue 项目生成完成后需通过以下验证：

- `npm install` 成功安装所有依赖（无报错）
- `npm run build` 成功构建生产版本（无 Vite/Rollup 报错）
- 构建产物 dist 目录包含完整的静态文件
- 页面在浏览器中正常渲染，路由跳转正常

常见构建失败原因及预防：

- 引用了不存在的本地资源文件——使用在线 URL 替代
- 组件 import 路径拼写错误——创建文件后立即在路由或父组件中正确引用
- package.json 依赖版本不兼容——使用经过验证的稳定版本组合
- vite.config.js 配置错误——直接使用参考配置，不做额外修改
